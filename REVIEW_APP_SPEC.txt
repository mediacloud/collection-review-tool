Review App Prototype – Specification (MediaCloud API client)
1. Purpose / Scope
Goal: A small, standalone web app that lets a manager seed a multi-collection `ReviewProject`, split the work into reviewer queues, and export the resulting decisions as a single aggregated CSV for ingestion by the existing MediaCloud web-search app.
Key behaviors:
Home page accepts one or more MediaCloud `collection_id`s (comma-separated).
Step 1: start a `ReviewProject`:
- Create the project and fetch deduped sources across all seed collections (stored at the project level, including stored seed collection names).
Step 2: on the project page, enter `queue_count` to generate reviewer queues:
- Partition the deduped source set into disjoint reviewer queues.
Reviewers open queue links and step through sources:
- Mark sources with KEEP/REMOVE decisions, and can propose new sources to ADD.
Queue completion is automatic:
- A queue is completed when exhausted (no undecided items remain); no manual “complete” action is required.
At the end, managers download the project CSV:
- Export aggregates the union of KEEP + ADD decisions across all reviewer queues.
Out of scope:
Any automated write-back to MediaCloud (this app only reads via the client and outputs CSV).
Complex auth/permissions integration with MediaCloud users.
Comments, multi-stage approvals, or real-time collaboration.
2. Stack & High-Level Architecture
Backend: Flask + SQLAlchemy
DB: SQLite (dev) with easy switch to Postgres (prod).
Responsibilities:
Managing review project state, reviewer queue state, and derived completion (computed on read).
Using the MediaCloud Python API client to fetch sources for each seed collection, dedupe them, and persist them for later queue splitting.
Serving JSON APIs to the frontend.
Generating CSV exports.
Frontend: Svelte SPA
Talks to the Flask JSON API.
Provides:
ReviewProject creation + a listing of existing projects on `/`.
A manager view for a specific `ReviewProject` on `/review-projects/:guid` (generate queues + download project CSV).
A reviewer queue workflow on `/reviews/:queue_guid` (step through sources and record decisions/proposals).
CSV download via the project page (aggregated project CSV).
3. Integration with MediaCloud (via API client)
3.1 Library
Use the official MediaCloud Python API client:
https://github.com/mediacloud/api-client
Install in the backend (e.g., pip install mediacloud or as per repo instructions).
3.2 Auth
Configure an API key/token via environment variable, e.g. MEDIACLOUD_API_KEY.
In Flask app startup:
Create a MediaCloud API client instance with that key.
For the prototype, assume this key has sufficient read permissions to fetch sources for the relevant collections.
3.3 Fetching sources for seed collections
Use the API client to fetch all sources in a given collection by collection_id.
Exact method / call will depend on the client, but the interface should look roughly like:
  # pseudo-code – agent should replace with actual client call  mc = MediaCloud(api_key)  sources = mc.sources_in_collection(collection_id)
The important part: for each returned source we need at least:
source_id (MediaCloud’s source id)
label or name
homepage (URL)
Seed ReviewProjectSource rows from this list:
source_id = source['id']
source_label = source['label'] or source['name']
source_homepage = source['homepage']
source_metadata = full source payload (stored for later CSV export)

Decision state is not assigned at this stage; reviewer queues (Step 2) create ReviewItem rows later with `decision='undecided'`.
4. Data Model
4.1 ReviewProject (manager-level container)
id (PK, int)
guid (UUID string, unique, required)
collection_ids (list of ints) – seed MediaCloud collection IDs
collection_names (list of strings) – seed collection names for display
name (string, optional), notes (text, optional)
guidelines_template (string, optional), edit_metadata (bool)
derived status: computed on read from reviewer queues (queue exhaustion)

4.2 ReviewProjectSource (deduped seed source)
id (PK, int)
review_project_id (FK → ReviewProject)
source_id (int) – deduplication key across all seed collections
source_label (string, optional)
source_homepage (string, optional)
source_metadata (text/json payload for later CSV export)

4.3 Review (reused as a reviewer queue)
id (PK, int)
review_project_id (FK → ReviewProject)
queue_guid (UUID string, unique, required)
queue_index (int) – stable bucket index for UI/linking
guidelines_template, edit_metadata
queue status is derived on read: completed when there are no undecided ReviewItems

4.4 ReviewItem (a source row inside a reviewer queue)
id (PK, int)
review_id (FK → Review queue)
source_id (int, nullable) – MediaCloud source ID (null for proposed-new ones)
source_label (string, optional)
source_homepage (string, optional)
is_new_source (bool, default false)
decision (enum string): undecided, keep, remove, add
decided_at (datetime, nullable)
source_metadata (json/text payload for later CSV export)

4.5 User (optional)
Can be omitted for v0; treat this as a single-user internal tool, or front it with simple basic auth/reverse-proxy.
5. Backend API Endpoints
All JSON except CSV export.

5.1 ReviewProject (Step 1 + Step 2 + manager read)
POST /api/review-projects/start
Body:
    { "collection_ids": [123, 456], "guidelines_template": "default", "edit_metadata": false, "name": "Optional project name" }
Behavior:
- Step 1 creates a ReviewProject
- Fetches MediaCloud sources for each seed collection_id
- Deduplicates sources across seed collections by source_id
- Stores deduped sources as ReviewProjectSource rows (including stored seed collection names)

POST /api/review-projects/<project_guid>/queues
Body:
    { "queue_count": 3 }
Behavior:
- Step 2 partitions the project's deduped sources into disjoint reviewer queues
- Creates Review + ReviewItem rows with decision='undecided'

GET /api/review-projects
Lists all ReviewProjects with derived status + aggregated queue stats.

GET /api/review-projects/<project_guid>
Returns project details plus per-queue summaries (derived on read).

GET /api/review-projects/<project_guid>/export
Exports a single aggregated project CSV (KEEP + ADD union across queues; non-blocking even if queues are not “completed”).

5.2 Reviewer queues (GUID-based)
GET /api/review-queues/<queue_guid>
Returns queue details + derived status (completed when exhausted).

GET /api/review-queues/<queue_guid>/guidelines
Returns rendered guidelines for the queue.

GET /api/review-queues/<queue_guid>/items
Query params:
page, page_size (optional)
decision (optional filter: undecided, keep, remove, add)
Response:
items array and total count.

POST /api/review-queues/<queue_guid>/items/<item_id>/decide
Body:
    { "decision": "keep" | "remove" | "add" }
Behavior:
Updates decision and decided_at.

POST /api/review-queues/<queue_guid>/items
Body:
    { "source_label": "New Outlet", "source_homepage": "https://new-outlet.example" }
Behavior:
Creates a new proposed source item inside the queue (is_new_source=true, decision='add').

6. Frontend (Svelte) UX
6.1 Home (/)
Input: comma-separated MediaCloud `collection_id`s.
Button: Start ReviewProject.
On submit:
POST /api/review-projects/start
On success:
navigate to /review-projects/:guid.

6.2 ReviewProject manager page (/review-projects/:guid)
Shows seed collection names, project stats, and derived project status.
If queues do not exist:
enter `queue_count` and call POST /api/review-projects/<guid>/queues.
If queues exist:
show queue cards (with progress) and Open Queue links.
Download Project CSV is available on the project page.

6.3 Reviewer queue (/reviews/:queue_guid)
On load:
GET /api/review-queues/:queue_guid
GET /api/review-queues/:queue_guid/items?decision=undecided&page=1
UI:
Header with queue progress + queue status.
Current undecided item viewer:
Show source_label and source_homepage (link).
Buttons:
Keep/Remove update the current item decision and then load next undecided.
Propose new source form creates an “add” item in the queue.
Queue completion is automatic:
when the queue has no undecided items, it is exhausted and the UI shows “Queue exhausted”.
7. Configuration & Ops
Env vars:
DATABASE_URL
MEDIACLOUD_API_KEY
On startup:
Instantiate a MediaCloud API client with MEDIACLOUD_API_KEY.
Error handling:
Surface simple, clear JSON errors when:
MC client fails (bad key, network).
Collection ID not found / no sources.
